## 0. 요약

이번 대회는 특정 풍력단지의 시간 단위 발전량을 예측하는 문제이지만,
성패는 단순히 MAE를 줄이는 데서 끝나지 않는다.

핵심은 다음 네 가지다.

1. **예측 기준시점 준수**
   D-1 14:00 KST 이후 생성·공개·확정된 정보는 사용할 수 없다.

2. **3개 KPX 그룹별 용량과 label 기간 차이 반영**
   Group 1/2는 2022-2024 label, Group 3은 2023-2024 label로 판독된다.

3. **정산금 관점의 평가 최적화**
   리더보드는 `1-NMAE`와 정산금획득률 계열 지표를 함께 반영한다.

4. **재현성과 규정 준수**
   외부 데이터, 사전학습모델, API 사용 제한, 코드 재현성이 2차 검증의 핵심 리스크다.

따라서 프로젝트는 `전략 PRD → 데이터·모델링 설계서 → 운영 마스터플랜` 순서로 진행한다.
이 문서는 첫 번째 산출물이며, 공식 데이터 공개 전의 가정과 확정 사실을 분리하는 것이 목적이다.

## 1. 근거 관리 원칙

### 1.1 근거 등급

| 등급 | 의미 | 예시 |
|------|------|------|
| E0 | 공식 대회 안내 또는 규칙 | Dacon 대회 페이지, 대회 규칙, 제출 안내 |
| E1 | 사전 워크숍 발표 자료/사진 | Dacon 운영 설명 슬라이드, GS E&R 세미나 사진 |
| E2 | 제공 데모 파일 | `chronos_wind_power_demo.ipynb` |
| E3 | 계산/추론 | 설비용량과 터빈 정격용량으로 터빈 수 검산 |
| E4 | 외부 리서치 | Chronos, AutoGluon, NWP, 시장/제도 조사 |

### 1.2 상태 라벨

| 상태 | 의미 | 문서화 방식 |
|------|------|-------------|
| 확정 | 근거가 명확하고 충돌이 없음 | 그대로 요구사항에 반영 |
| 판독 불확실 | 사진이 흐릿하거나 문구가 일부 불명확 | 단정하지 않고 보수 표현 사용 |
| 공식 공개 후 재검증 | 데이터 공개 전에는 단정 위험 | 검증 체크리스트에 포함 |
| 추론 | 근거값에서 계산한 2차 정보 | 반드시 `추론`으로 표시 |

### 1.3 환각 방지 규칙

- 사진에서 안 보이는 컬럼명을 임의로 만들지 않는다.
- 데모 노트북의 synthetic 데이터 구조를 실제 대회 데이터 구조로 승격하지 않는다.
- 외부 리서치로 얻은 일반 지식을 대회 규칙으로 단정하지 않는다.
- 평가식 표기 충돌(`FICR`, `SCR`, `정산금획득률`)은 공식 평가 페이지와 데이터 공개 후 다시 확인한다.
- 모든 외부 데이터 후보는 `생성 시각`, `공개 시각`, `수집 시각`, `라이선스`, `재현 경로`를 남긴다.

## 2. 문제 정의

### 2.1 대회 과제

기상청 기상예보 데이터와 풍력발전량 데이터를 활용하여,
특정 풍력단지의 향후 시간 단위 발전량을 예측하는 AI 모델을 개발한다.

참가자는 제공된 학습 데이터를 기반으로 예측 모델을 만들고,
평가 기간에 해당하는 시간 단위 KPX 그룹별 발전량 예측값을 CSV로 제출한다.

### 2.2 예측 단위

| 항목 | 정의 | 상태 |
|------|------|------|
| 예측 대상 | `KPX Group 1`, `KPX Group 2`, `KPX Group 3` 발전량 | 확정 |
| 시간 해상도 | 시간 단위 | 확정 |
| 제출 행/컬럼 | `일시`, `KPX그룹_1 발전량`, `KPX그룹_2 발전량`, `KPX그룹_3 발전량` 형태로 판독 | 공식 공개 후 재검증 |
| 제출 단위 | 발전량 값. MW/MWh 등 세부 단위는 sample submission 공개 후 확정 | 공식 공개 후 재검증 |
| 평가 연도 | 2025년 평가 데이터 | 확정 |

### 2.3 제약의 본질

풍력발전량은 풍속, 풍향, 기온, 기압, 습도, 난류, 지형, 터빈 상태의 영향을 받는다.
특히 태백가덕산풍력발전은 산악 능선과 복잡 지형에 놓여 있어,
격자 예보를 그대로 사용하는 것만으로는 발전량 곡선을 충분히 설명하기 어렵다.

대회 문제는 다음 세 문제가 결합된 형태다.

1. **기상예보 후처리 문제**
   LDAPS/GFS 격자 예보를 터빈 허브 높이와 단지 지형에 맞게 해석해야 한다.

2. **발전량 변환 문제**
   풍속·풍향·온도 등 예보 변수를 그룹별 발전량으로 변환해야 한다.

3. **운영/정산 관점 최적화 문제**
   평균 오차뿐 아니라 정산금 획득률 구간을 의식한 예측이 필요하다.

## 3. 이해관계자와 성공 기준

| 이해관계자 | 관심사 | 프로젝트 요구사항 |
|------------|--------|-------------------|
| 참가팀 | 높은 Private 점수, 2차 검증 통과, 발표 경쟁력 | 재현 가능한 파이프라인, 실험 추적, 규정 준수 |
| 발전사업자 | 예측 오차 감소, 정산금 확보, 운영 안정성 | 기상·터빈 특성을 반영한 설명 가능한 모델 |
| 전력계통/시장 | 변동성 대응, 수급 안정화 | 예측 기준시점과 제출 현실성을 반영한 모델 |
| 데이콘 운영 | 공정성, 부정행위 방지, 재현성 검증 | 외부 데이터/모델/코드 근거 제출 |

### 3.1 1차 성공 기준

- Private 리더보드 상위권 진입
- 제출 CSV 형식 오류 0건
- 일별 제출 횟수와 최종 선택 파일 관리
- 데이터 누수 의심 요소 0건

### 3.2 2차 성공 기준

- Private Score 오차 범위 내 복원 가능
- train/inference 코드 분리
- 외부 데이터 사용 근거와 재현 코드 제출 가능
- 문제 이해, 데이터 분석, 성능 개선, 적용 가능성을 발표 자료에서 설명 가능

## 5. 평가 지표 해석

### 5.1 지표 구성

워크숍 사진 기준 리더보드는 다음 점수를 제공한다.

| 지표 | 의미 | 방향성 | 상태 |
|------|------|--------|------|
| `1-NMAE` | 설비용량 기준 정규화 평균절대오차를 1에서 뺀 값 | 높을수록 좋음 | 확정 |
| `FICR` 또는 `SCR` | 정산금획득률 계열 지표 | 높을수록 좋음 | 표기 재검증 |
| Total Score | `1-NMAE`와 정산금 지표 결합 | 높을수록 좋음 | 확정 |

### 5.2 NMAE

워크숍 사진 기준 그룹별 NMAE는 다음처럼 해석된다.

```text
NMAE_g = mean(abs(y_pred - y_true) / capacity_g)
1-NMAE = 1 - mean(NMAE_g for 3 groups)
```

공식 안내의 일반 nMAE 설명은 다음과 같다.

```text
nMAE = (1 / 설비용량) * mean(abs(y_i - yhat_i))
```

### 5.3 정산금획득률

워크숍 Dacon 슬라이드 기준으로 시간별 NMAE 구간에 따라 정산금이 산정된다.

| 시간별 NMAE 구간 | 정산 단가 | 상태 |
|------------------|-----------|------|
| 6% 이하 | 4원/kWh | 워크숍 근거 |
| 6% 초과, 8% 이하 | 3원/kWh | 워크숍 근거 |
| 8% 초과 | 0원/kWh로 해석 | 추론 |

주의: GS E&R 시장제도 설명 슬라이드에는 일반 제도 맥락의 다른 구간이 등장한다.
대회 점수 산식에는 Dacon 운영 설명 슬라이드 기준을 우선 적용하되,
공식 평가 페이지와 데이터 공개 후 반드시 재확인한다.

### 5.4 유효 평가 시간

워크숍 사진 기준, 실제 발전량이 해당 그룹 설비용량의 10% 이상인 시간만 평가에 반영된다.

| 그룹 | 설비용량 | 10% 기준 |
|------|----------|----------|
| Group 1 | 21.6MW | 2.16MW |
| Group 2 | 21.6MW | 2.16MW |
| Group 3 | 21.0MW | 2.10MW |

위 기준값은 설비용량에서 계산한 추론값이다.
실제 평가 구현에서는 label 기준과 단위가 공식적으로 공개되어야 정확히 재현할 수 있다.

### 5.5 전략적 함의

- 야간/저풍속/정지 상태 오차가 전체 MAE에는 영향을 주더라도,
  유효 평가 시간 필터 때문에 점수 기여가 다를 수 있다.
- 정산금 구간은 오차를 `6%`, `8%` 임계값 근처에서 관리하는 전략을 요구한다.
- Total Score가 `1-NMAE 50% + 정산금 지표 50%`라면,
  평균 오차 최소화 모델과 정산 구간 최적화 모델을 함께 비교해야 한다.

## 8. 시장/제도 맥락

재생에너지 발전량 예측은 단순 연구 문제가 아니라 전력계통 안정성과 정산금 확보에 연결된다.
풍력은 태양광보다 변동성이 크고, 복잡 지형에서는 예보 오차와 지형 효과가 커진다.

워크숍 메모 기준으로 태양광 예측 오차는 약 3-6%, 풍력은 약 10-12% 수준으로 언급되었고,
풍력 예측 목표 오차는 약 8% 수준으로 제시되었다.
이 숫자는 시장/문제 난이도 맥락으로만 사용하고,
대회 공식 목표나 평가 기준으로 단정하지 않는다.

이 대회의 차별점은 `평균 예측오차율`과 `정산금획득률`을 함께 반영한다는 점이다.
따라서 모델 개발은 다음 두 관점을 동시에 가져야 한다.

1. 발전량 곡선 자체를 잘 맞추는 회귀 문제
2. 정산 인센티브 구간 안으로 오차를 넣는 운영 최적화 문제

## 9. Perplexity/외부 리서치 활용 계획

Perplexity 또는 웹 리서치는 대회 데이터 사실을 확정하는 용도가 아니라,
모델링 아이디어와 시장/제도 배경을 넓히는 용도로만 사용한다.

### 9.1 우선 리서치 질문

| 우선순위 | 질문 | 산출물 반영 위치 |
|----------|------|------------------|
| P0 | 풍력발전 단기 예측에서 NWP post-processing의 실무 표준은 무엇인가? | 데이터·모델링 설계서 |
| P0 | 복잡 지형 풍력 예측에서 grid pooling, terrain feature, hub-height wind 보정은 어떻게 쓰이는가? | 피처 설계 |
| P1 | Chronos/Chronos-Bolt/Chronos-2가 외생 기상변수 풍력 예측에 적합한가? | 모델 후보 비교 |
| P1 | 예측 정산금 최적화에서 threshold-aware loss나 calibration 사례가 있는가? | 평가 최적화 |
| P2 | 국내 재생에너지 발전량 예측제도 변경 흐름은 무엇인가? | 발표 배경 |

### 9.2 외부 리서치 검증 규칙

- 블로그/기사보다 논문, 공식 문서, 모델 카드, 기관 보고서를 우선한다.

- 원격 API 호출이 필요한 방법은 구현 후보에서 제외한다.
- 유료/비공개 데이터가 필요한 방법은 외부 데이터 후보에서 제외한다.
