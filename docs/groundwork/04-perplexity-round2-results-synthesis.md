## 6. R2-4 Chronos-2 공식성 검증

R2-4 결과는 Chronos-2를 이전보다 강한 P1 후보로 올릴 수 있는 근거를 제공한다.
다만 P0 주력은 여전히 LightGBM/CatBoost 기반 tabular NWP baseline이다.

| 모델 | 적격성 | 판단 |
|------|--------|------|
| Chronos-2 | 2025-10-30 HF initial commit, Apache 2.0, 로컬 실행, native covariate 지원 | P1 실험 후보 |
| Chronos-Bolt | 2024년 공개, Apache 2.0, 로컬 실행 가능, native covariate 미지원 | P2 zero-shot 또는 보조 baseline |
| Chronos T5 계열 | 2024년 공개, Apache 2.0, 로컬 실행 가능, covariate 미지원 | 보류 |
| AutoGluon-TimeSeries v1.5 | 2025-12-19 공개, Chronos-2 orchestration 가능 | P1/P2 실험 도구 후보 |

설계 반영:

- `model_registry`에 `amazon/chronos-2`, 공개일 `2025-10-30`, 라이선스 `Apache-2.0`, 로컬 실행 가능 여부를 기록한다.
- SageMaker, AutoGluon Cloud, Hugging Face Inference API 등 원격 추론 경로는 사용하지 않는다.
- Chronos-2는 NWP future covariate를 직접 반영할 수 있는 후보로 보되, 실제 성능은 local CV로만 판단한다.
- Chronos/Bolt는 native covariate가 없으므로 BARAM NWP 구조의 1차 주력으로 두지 않는다.

## 7. R2-5 LDAPS/GFS 공식 사양 검증

### 7.1 가장 중요한 변경점

기상청은 2026-03-31 UM 기반 자료 제공을 종료하고,
2026-04부터 KIM 단독 현업 운영으로 전환했다는 공식 근거가 확인되었다.
따라서 BARAM 데이터가 KMA 일반 LDAPS 자료 흐름을 반영한다면,
학습/평가 기간 일부에 UM-LDAPS와 KIM-LDAPS 사양이 혼재할 가능성을 고려해야 한다.

| 항목 | 공식 일반 사양 | 대회 적용 |
|------|----------------|-----------|
| LDAPS 모델 기반 | UM 기반에서 KIM 단독 운영으로 전환 | 대회 파일 메타로 확인 전 미확정 |
| LDAPS 수평 해상도 | UM: 1.5km, KIM: 1km | 파일 헤더/격자 정의 확인 필요 |
| LDAPS 연직층 | UM: L70, KIM: L40 | 레코드/레벨 목록 확인 필요 |
| LDAPS 예보 주기 | 1일 4회 00/06/12/18 UTC가 유력 | 파일명 기준시각 확인 |
| GFS 출력 격자 | 0.25도 pgrb2 공식 인벤토리 존재 | 대회 제공 파일 crop/서브셋 여부 확인 |
| GFS 80m/100m 바람 | 공식 인벤토리에 UGRD/VGRD 존재 | 대회 파일 포함 여부 확인 |

### 7.2 데이터 공개 후 NWP 메타데이터 체크

| 체크 항목 | 이유 |
|-----------|------|
| 파일 포맷, 확장자, magic bytes | GRIB2/CSV/NetCDF 여부 확인 |
| base time, valid time, lead hour | h016~h039와 cutoff 검증 |
| forecast source별 grid_id, lat/lon, projection | LDAPS 16/GFS 9 판독 검증 |
| 변수 목록과 단위 | KMA/NOAA 전체 사양과 대회 서브셋 분리 |
| 높이 레벨 | 10m/80m/100m 바람 사용 가능성 확인 |
| LDAPS 모델 전환 경계 | UM/KIM 혼재로 인한 분포 shift 가능성 확인 |
| train/test 기간별 forecast model version | 2025 평가 데이터와 2022-2024 train의 사양 차이 확인 |

## 8. R2-6 KPX 정산제도와 Dacon 평가 산식 분리

R2-6의 핵심 결론은 간단하다.
KPX 실제 제도는 발표 배경으로 유용하지만, Dacon 평가 산식으로 자동 적용하면 안 된다.

| 항목 | Dacon 대회 기준 | KPX 실제 제도 | 모델링 반영 |
|------|-----------------|---------------|-------------|
| 정산금획득률 정의 | 공식 평가 페이지 정의 우선 | 제도 배경 | Dacon 정의 고정 |
| Public/Private 세부 기준 | 추후 공개 | 해당 없음 | 재검증 |
| 워크숍 6%/8% | 대회 운영 슬라이드 판독 | KPX 구제도와 유사 | 시나리오 A로 보관 |
| KPX 4%/6% | Dacon 기준 아님 | 2025/2026 제도 변화 후보 | 발표 배경, 시나리오 B 참고용 |
| KPX 예측 제출 마감 | Dacon cutoff와 다름 | D-1 10시/17시 등 제도 맥락 | 대회 cutoff에 적용 금지 |

금지할 혼용:

- KPX 4%/6% 임계값을 BARAM local simulator 공식값으로 고정
- KPX 설비이용률 기반 계산을 Dacon `이론상최대정산가능금액`으로 대체
- KPX 제출 마감을 BARAM `D-1 14:00 KST` cutoff 대신 사용
- KPX 신제도 단가 구조를 대회 정산금획득률에 직접 삽입

## 11. Perplexity 후속 세션 운영 가이드

사용자가 Perplexity 안내를 받을 때는 아래 기준으로 "기존 세션에서 이어서 할지"와
"새 세션을 팔지"를 함께 안내한다.

| 상황 | 권장 | 이유 |
|------|------|------|
| 같은 R2-x 세션의 답변을 표로 재정리하거나 출처 URL만 보강 | 기존 세션 | 같은 문맥과 출처 추적을 유지하는 편이 좋음 |
| 같은 claim의 문구를 더 보수적으로 바꾸거나 누락된 출처를 확인 | 기존 세션 | claim lineage가 끊기지 않음 |
| R2-1 평가 산식에서 R2-6 KPX 제도로 넘어가는 등 주제가 바뀜 | 새 세션 | 공식 평가 산식과 외부 제도 맥락이 섞이는 것을 방지 |
| 공식 페이지/FAQ가 업데이트된 뒤 같은 질문을 다시 검증 | 새 세션 | 날짜별 증거를 분리해야 함 |
| 다른 모드/모델을 써서 재검증할 때 | 새 세션 | 답변 품질 비교와 출처 차이 추적이 쉬움 |
| 대회 공식 데이터 파일 또는 스키마를 다룰 때 | 새 세션, 단 공식 업로드 허용 확인 후 | 파일 업로드 위험과 세션 목적을 분리해야 함 |
| Perplexity가 추론과 공식 사실을 섞기 시작했을 때 | 새 세션 | 오염된 맥락을 끊고 시스템 지침을 다시 고정 |

기본 규칙:

- Space는 계속 `BARAM 2026 Wind Research`를 사용한다.
- 세션명은 `R2-<번호> <주제> YYYY-MM-DD`처럼 날짜를 붙인다.
- 결과를 문서에 반입할 때는 세션명, 날짜, 모드, 모델, 업로드 파일을 반드시 기록한다.
- 공식 데이터 파일은 외부 업로드 규칙이 확인되기 전까지 Perplexity에 올리지 않는다.
- 후속 질문이 "출처 보강"인지 "새 검증"인지 먼저 판단하고, 새 검증이면 세션을 새로 판다.
