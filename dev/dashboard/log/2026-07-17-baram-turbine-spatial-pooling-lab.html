완료 범위

  • 공식 info.xlsx의 17개 터빈 위·경도와 설비 용량을 정규화해 임의 좌표 없이 LDAPS 16개·GFS 9개 grid와 직접 연결했다.
  • 각 터빈에서 haversine 거리를 계산하고 nearest와 IDW를 적용한 뒤, 발전단지 안에서는 터빈 용량 비율로 다시 집계하는 결정론적 공간 pooling 계약을 구현했다.
  • 부분 grid 결측은 관측 가능한 weight만 재정규화하고 전체 grid 결측은 NaN으로 보존했다. imputer는 train에만 fit하고 holdout·test에는 transform만 적용했다.
  • 공식 mean, expanded, vector 통제군과 공간 후보를 동일 RandomForest·seed·2024 time holdout·target별 non-null mask·metric·prediction clipping 조건에서 비교했다.
  • 모델 파일, registry CSV, submission CSV와 outputs/ 산출물은 만들지 않았으며 기존 baseline·registry schema도 변경하지 않았다.

실패 사례·한계

공간 해상도 한계: GFS는 17개 터빈 모두 같은 grid 5를 nearest로 선택해 nearest만으로는 터빈 위치를 구분하지 못했다. LDAPS의 VESTAS-01은 1·2순위 grid 거리 차가 약 18m에 불과해 좌표·격자 경계에 민감하다. 또한 best 후보도 Group 3의 8% 안착률은 0.364871→0.364433으로 0.000438 하락했다.
  • LDAPS test의 47개 forecast-variable 쌍은 16개 grid가 모두 결측이었다. 공간 pooling으로 복원하지 않고 NaN을 보존한 뒤 train-fit median만 적용했다.
  • IDW p=1도 vector보다 개선됐지만, p=2보다 total·FICR와 target 개선 폭이 낮아 채택하지 않았다.
  • all-group 550개 후보는 다른 발전단지의 공간 요약도 사용하므로, 396개 own-group 후보를 차원 증가 효과를 분리하는 통제군으로 유지했다.
  • actual은 원본 값을 유지했고 prediction만 0~capacity로 clip했다. Group 3 결측 label은 0으로 대체하지 않고 target별 non-null mask를 사용했다.

Decision Box

채택 후보 — all-group IDW p=2: 공식 터빈 좌표로 계산한 거리 가중 pooling 231개를 vector 319개에 더한다. 2024 holdout에서 total score가 0.586482→0.592485로 가장 높고 모든 target의 NMAE·FICR·6% 안착률이 개선됐다. 다만 GFS 해상도, LDAPS 경계 민감도와 Group 3 8% 퇴행 때문에 자동 registry 승격은 하지 않고, own-group nearest 396개를 더 작은 강건성 대안으로 함께 유지한다.
