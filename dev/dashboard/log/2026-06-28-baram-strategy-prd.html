전략 PRD — 데이터 팩트 레저 포함

2026-06-28 KST · docs/baram-strategy-prd

완료

  • 프로젝트 README와 문서 인덱스를 생성했다.
  • 전략 PRD를 작성하고 문제 정의, 이해관계자, 평가식, 규정 리스크를 정리했다.
  • 사전 워크숍 사진에서 판독한 KPX 그룹, 터빈 수, 용량, 터빈 모델, 데이터 기간, LDAPS/GFS/SCADA, 예측 기준시점 정보를 데이터 팩트 레저로 분리했다.
  • 지명은 불확실 처리하고 색상 기반 그룹-터빈 매핑만 확정 사실로 반영했다.
  • 프로젝트 대시보드와 루트 대시보드 프로젝트 카드를 갱신했다.

검증

  • 문서 내 데이터 항목을 확정, 판독 불확실, 공식 공개 후 재검증, 추론으로 구분했다.
  • 데모 노트북의 synthetic 데이터 구조를 실제 대회 데이터로 단정하지 않도록 명시했다.
  • 평가 지표 표기 충돌(FICR/SCR)은 재검증 항목으로 남겼다.

다음

  • 데이터·모델링 설계서 작성: 공식 데이터 공개 직후 스키마 검증표, 피처 설계, validation split, baseline metric 재현 설계.
  • 필요 시 사용자와 대화하며 불확실한 사진 판독 항목을 추가 확인한다.
